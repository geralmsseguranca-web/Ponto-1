⚙️ Assistente de Configuração

Configure o Firebase e publique no GitHub Pages em minutos

🔥 Passo 1 de 5  ·  Firebase
Criar projeto no Firebase
O Firebase vai guardar todos os dados na nuvem — colaboradores, registros de ponto e usuários — sincronizados em todos os dispositivos em tempo real.

📋 Siga estes passos:

  • 1Acesse console.firebase.google.com e faça login com sua conta Google
  • 2Clique em "Criar um projeto" (ou "Add project")
  • 3Dê um nome ao projeto, ex: grupo-ms-ponto
  • 4Desative o Google Analytics (não é necessário) e clique "Criar projeto"
  • 5Aguarde a criação e clique "Continuar"
✅ O projeto Firebase é gratuito no plano Spark — suficiente para uso diário com colaboradores.
🗄️ Passo 2 de 5  ·  Banco de Dados
Ativar o Realtime Database
O Realtime Database é o banco de dados que vai armazenar os registros de ponto em tempo real.

📋 No console do Firebase:

  • 1No menu lateral esquerdo, clique em "Build" → "Realtime Database"
  • 2Clique em "Create Database"
  • 3Selecione a localização: escolha "United States" (padrão) e clique Next
  • 4Em "Security rules", selecione "Start in test mode" e clique "Enable"

🔒 Configurar regras de segurança:

  • 5Após criar, clique na aba "Rules" no Realtime Database
  • 6Substitua o conteúdo pelas regras abaixo e clique "Publish":
{ "rules": { ".read": true, ".write": true } }
⚠️ Essas regras permitem acesso aberto — adequado para uso interno. Para maior segurança futura, consulte a documentação do Firebase Authentication.

  • 2Dê um apelido ao app, ex: ponto-web, e clique "Register app"

💡 Após preencher e salvar, o arquivo index.html será atualizado automaticamente com suas credenciais.
🐙 Passo 4 de 5  ·  GitHub Pages
Publicar no GitHub Pages
O GitHub Pages hospeda seu sistema gratuitamente com um link fixo que você pode enviar para os colaboradores. Não precisa pagar hospedagem.

📋 Criar conta e repositório:

  • 1Acesse github.com e crie uma conta gratuita (ou faça login)
  • 2Clique em "New repository" (botão verde no canto superior direito)
  • 3Nome do repositório: ponto-ms — deixe como Public e clique "Create repository"

📤 Fazer upload do arquivo:

  • 4Na página do repositório criado, clique em "uploading an existing file"
  • 5Arraste o arquivo index.html (este arquivo, já com as credenciais salvas) para a área de upload
  • 6Clique em "Commit changes" para salvar

🌐 Ativar o GitHub Pages:

  • 7Vá em Settings (engrenagem no menu do repositório)
  • 8No menu lateral, clique em "Pages"
  • 9Em "Branch", selecione main e clique "Save"
  • 10Aguarde ~1 minuto. Seu link aparecerá no formato:
    https://seu-usuario.github.io/ponto-ms/
🎉 Passo 5 de 5  ·  Pronto!
Sistema pronto para uso!
Configure o Firebase no arquivo e publique no GitHub. O sistema funcionará com dados sincronizados na nuvem.

✅ Resumo do que foi feito:

  • ✓Firebase: banco de dados em nuvem criado e configurado
  • ✓GitHub Pages: hospedagem gratuita com link fixo
  • ✓Login: acesso admin padrão — usuário admin senha admin123

📱 Como enviar acesso para colaboradores:

  • 1Entre no sistema como admin e vá na aba 🔑 Usuários
  • 2Crie um acesso para cada colaborador com usuário e senha
  • 3Envie o link do GitHub Pages + usuário + senha para cada um
⚠️ Lembre-se de trocar a senha do admin padrão após o primeiro acesso!
